01 — The Imperative

Agentic AI rewrites the threat model of the developer inner loop

AI coding agents now read source, execute commands, install packages, call internal APIs, and open pull requests autonomously. Run on unmanaged laptops and ad-hoc VMs, they multiply both our velocity and our blast radius. The environment is now a control plane — and it is not yet under control.

⚠

Autonomy without containment

An agent that can run arbitrary code and reach the internet is an insider with super-human speed. Without a sandbox, prompt injection becomes remote code execution and data exfiltration.

⌘

Secrets sprawl on endpoints

Long-lived tokens, cloud keys and source live on thousands of laptops and one-off VMs. Each is an unmanaged copy of our crown jewels and an audit blind spot.

⟳

"Works on my machine" at scale

Four major stacks (Java, .NET, Python, JavaScript) × thousands of devs = unbounded environment drift, slow onboarding, and irreproducible builds that frustrate audits and incident response.

◎

No identity for non-humans

AI agents and automation act with borrowed human credentials. We cannot attribute, scope, throttle, or revoke their access — a governance and compliance gap regulators increasingly probe.

◷

Speed pressure is real

Competitors are shipping agent-assisted delivery today. Banning the tools cedes the advantage; allowing them ungoverned courts the next headline breach. We need a third path.

§

Regulatory gravity

SBOM mandates, supply-chain integrity (SLSA), EU AI Act and sector rules expect provenance and control over how — and by what — code is produced. Our inner loop must become evidence-grade.

"The most privileged actor in our software supply chain is no longer a person. It's an agent running on a machine we don't manage."

01 — The Imperative

Agentic AI rewrites the threat model of the developer inner loop

AI coding agents now read source, execute commands, install packages, call internal APIs, and open pull requests autonomously. Run on unmanaged laptops and ad-hoc VMs, they multiply both our velocity and our blast radius. The environment is now a control plane — and it is not yet under control.

⚠

Autonomy without containment

An agent that can run arbitrary code and reach the internet is an insider with super-human speed. Without a sandbox, prompt injection becomes remote code execution and data exfiltration.

⌘

Secrets sprawl on endpoints

Long-lived tokens, cloud keys and source live on thousands of laptops and one-off VMs. Each is an unmanaged copy of our crown jewels and an audit blind spot.

⟳

"Works on my machine" at scale

Four major stacks (Java, .NET, Python, JavaScript) × thousands of devs = unbounded environment drift, slow onboarding, and irreproducible builds that frustrate audits and incident response.

◎

No identity for non-humans

AI agents and automation act with borrowed human credentials. We cannot attribute, scope, throttle, or revoke their access — a governance and compliance gap regulators increasingly probe.

◷

Speed pressure is real

Competitors are shipping agent-assisted delivery today. Banning the tools cedes the advantage; allowing them ungoverned courts the next headline breach. We need a third path.

§

Regulatory gravity

SBOM mandates, supply-chain integrity (SLSA), EU AI Act and sector rules expect provenance and control over how — and by what — code is produced. Our inner loop must become evidence-grade.

"The most privileged actor in our software supply chain is no longer a person. It's an agent running on a machine we don't manage."

02 — Current State

A strong foundation — with structural gaps for AI

We already run GitHub Enterprise, Azure and a Citrix VDI estate. The opportunity is to consolidate the inner loop into one governed environment — and retire the cost and risk of running it across laptops, VDI desktops and per-seat tooling.

Where today's setup breaks for agentic AI

  • Compute is fragmented. Laptops, personal VMs and inconsistent VDI images — no common, policy-enforced runtime for code or agents.
  • Egress is open. Dev machines reach the internet freely; an agent can pull a malicious package or post data anywhere.
  • Credentials are static. PATs and keys live on disk instead of short-lived, federated tokens minted per session.
  • Telemetry is thin. We can't reconstruct what an agent did, with what data, against which systems.
  • Onboarding is slow. Days of setup per stack per engineer; even longer for contractors and acquisitions.
  • The estate is expensive. Powerful laptops, a VDI farm and per-seat tooling licences scale linearly with every new developer.

03 — The Solution

One governed Cloud Development Environment

Move the inner loop off the endpoint and into ephemeral, policy-enforced workspaces in Azure — accessed through Citrix and the browser, wired into GitHub Enterprise, and built for AI agents from day one. Code and credentials never rest on a laptop again.

P1

Ephemeral by default

Workspaces are created on demand from versioned golden images and destroyed after use. No persistent state, no drift, no forensic ambiguity.

P2

Secure by design

Zero-trust access, default-deny egress, secrets injected just-in-time from Key Vault, full session telemetry to the SIEM. Security is the path of least resistance.

P3

Agent-ready

Every agent runs in a contained workspace with its own scoped, short-lived identity, an egress allowlist, and a model gateway that logs and filters every call.

P4

Golden-path productive

Pre-built, compliant stacks for Java, .NET, Python and JavaScript. A developer is coding in minutes — the secure way is the fast way.

05 — Security Model

Containing the agent's blast radius

Every agentic workflow is wrapped in five enforced controls. Even a fully compromised or hijacked agent stays inside a small, observable box.

01

Identity & least privilege

Each agent gets its own non-human identity with narrowly scoped, time-boxed tokens. No standing access, no shared service accounts, full attribution.

02

Isolation

Agents execute in sandboxed, ephemeral workspaces with no lateral path to other tenants, production, or the corporate network.

03

Egress control

Default-deny networking. Package registries and approved APIs are allowlisted through an inspecting proxy; everything else is blocked and logged.

04

Guardrails & gateway

A model gateway mediates every LLM call — logging prompts/responses, redacting secrets, and enforcing policy on tools the agent may invoke.

05

Observability & provenance

Every command, file touch and API call streams to Sentinel. Builds carry SBOM and SLSA provenance — we can prove what produced any artifact.

∑

Defense in depth

No single control is trusted alone. Identity, isolation, egress, guardrails and audit compound so a failure in one layer is caught by the next.

Maps directly to the controls our insurance regulators ask for

Aligned to the prudential technology and cyber requirements across our key markets — Hong Kong, Singapore, Malaysia, Indonesia and Australia.
