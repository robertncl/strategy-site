The starting point

Your software is mostly other people's code.

Every container you ship is a stack of someone else's base OS, language runtime, and open-source packages. That inherited surface is where the CVEs live — and where modern supply-chain attacks land. You're accountable for code you didn't write.

70–90%of a typical app is open-source & third-party code
~28k+new CVEs disclosed per year — and rising
3days median for an exploit to appear after disclosure

Log4Shell, SolarWinds, the xz backdoor — the perimeter moved into your build. Regulators noticed: SBOMs and provenance are now table stakes for the enterprise.

The problem

Your teams run on a CVE treadmill.

  • Bloated base images ship hundreds of packages you never call
  • Each one is a scanner finding someone must triage and patch
  • Engineers patch CVEs instead of building product
  • Audits stall on "where did this binary come from?"

Most of the vulnerabilities you remediate are in software you never run.

A full OS distro in a container means a shell, a package manager, and dozens of libraries that exist only to generate scanner noise — and widen the blast radius.

The idea

Don't out-patch the problem. Start from secure.

Shift security all the way left — to the artifact itself. Build on images and dependencies that are minimal by design, built from source, signed, and continuously rebuilt so CVEs are fixed upstream before they ever reach your scanners.

Honest about trade-offs

Risks & mitigations.

No shell to debug

Distroless removes the shell. Use the matching -dev image in CI and ephemeral debug containers in prod — a security win, not a blocker.

Migration effort

Most moves are a one-line FROM change. Start with high-volume bases for the biggest return; phase the long tail.

Vendor reliance

Images are standard OCI; libraries are standard formats. Mirror everything internally — no hard lock-in if you ever exit.

Cost vs. free base

The license is offset by reclaimed engineering hours, avoided breaches, and faster audits — quantified on the calculator slide.
