01 — Executive overview

Scale by autonomy, not by tool

Every agent is built on approved platforms and governed through one shared identity, data-protection and audit fabric. The programme is sequenced by autonomy — the safest, most defensible way to scale in a regulated environment. Each wave has an explicit promotion gate; nothing advances until the prior wave is stable, accurate and incident-free.

  1. Wave 1

    L1 Advise

    Read-only agents that summarise, explain and report. Immediate value, no change risk.

    Read-only
  3. Wave 2

    L2 Propose

    Agents draft tickets, PRs and pages for human approval. They create proposals but never make them effective.

    Create-only drafts
  5. Wave 3

    L3 Act-with-approval

    Agents execute reversible changes — merge, deploy, quarantine, rotate, contain — only after explicit human approval.

    Execute reversible change

02 — Operating model

Built once, reused by every agent

A central orchestrator receives an event, decomposes it, delegates to the right specialist, maintains run state, and returns one consolidated result. Specialists are single-purpose and independently governed. The orchestrator can never escalate autonomy — it only invokes the autonomy each specialist already holds.

Event / request webhook · schedule · chat
Orchestrator routes · maintains state · consolidates

The autonomy spine

L1

Advise

Read-only

✓ Allowed

Query (read), summarise, explain, prioritise, report, read-only dashboards, advisory notifications.

✕ Not allowed

Any write to a system of record.

Gate: none needed — read-only
L2

Propose

Create-only drafts

✓ Allowed

Everything in L1 + open draft PRs, file triage tickets, post PR comments, create draft/staging pages.

✕ Not allowed

Merge, deploy, publish controlled pages, close/approve, waive, rotate, quarantine, delete, config change.

Gate: human approves each proposal
L3

Act-with-approval

Execute reversible change

✓ Allowed

Everything in L2 + execute an action (merge, deploy, quarantine, rotate, contain).

✕ Not allowed

Unattended / irreversible action without approval.

Gate: per-action approval + full audit & rollback

Cross-cutting guardrails — every agent, every wave

  • 🆔

    One Entra Agent ID per agent

    Least-privilege, short-lived scoped tokens. Read-only by default; write/act scopes added only at L2/L3.

  • 📇

    Registered in Agent 365

    Accountable human owner; visible in the registry and Agent Map.

  • 🛡️

    Purview DLP on I/O

    Sensitivity labels honoured; no secrets, PII or MNPI in inputs or outputs.

  • ✅

    Allow-listed tools

    Connectors verified; untrusted tools sandboxed.

  • 📡

    All activity to Sentinel

    Full logging; shadow-agent discovery on.

  • 🧱

    Instruction / data separation

    Retrieved code, tickets and pages are treated as data, not instructions — embedded "instructions" are ignored.

  • 🙋

    Human-in-the-loop

    For every write (L2) and every act (L3).

  • ⚖️

    Governance gate before build

    Sponsor sign-off, security & privacy review, owner + Agent ID, monitoring rule, quarterly review — in the AI Risk register.

06 — Governance & risk

Control demonstrably maintained as autonomy grows

The consolidated risk register and RACI keep accountability explicit at every level. Blast radius is bounded by design: read → draft → approved-act.

Consolidated risk register

All waves

Indirect prompt injection via ingested content

Instruction/data separation; allow-listed tools; output DLP; bounded blast radius (read → draft → approved-act).

L2 / L3

Agent makes a change effective without approval

Branch protection blocks self-merge; no act scope at L2; JIT per-action approval at L3; "cannot-act" test.

L2 / L3

Proposal / alert noise

Confidence thresholds + volume caps; acceptance-rate KPI; continuous tuning.

L2 / L3

Attribution confusion (AI vs human)

Dedicated bot identity; labels + "pending review" notes; actor_is_agent.

All waves

Credential sprawl / over-scoping

One scoped credential per tool in Key Vault; periodic recertification; JIT act scopes.

All waves

Over-reliance on agent output

"Advise / propose, not decide"; mandatory citations; human ownership of all actions.

All waves

Scope creep across autonomy levels

Hard level definitions; promotion gates with fresh security review of new scopes.

All waves

Reviewer overload

CODEOWNERS routing; prioritise high-confidence; volume caps.
